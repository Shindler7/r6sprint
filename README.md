# Парсер логов

Задача: рефакторинг предоставленного кода проекта, обеспечение работы
функциональных возможностей.

## Функционал

Пусть есть логи:
System(requestid):

- trace
- error
  App(requestid):
- trace
- error
- journal (человекочитаемая сводка)

Есть прототип штуки, которая умеет:

- парсить логи
- фильтровать
  -- по requestid
  -- по ошибкам
  -- по изменению счёта (купить/продать)

Модель данных:

- Пользователь (userid, имя)
- Вещи
  -- Предмет (assetid, название)
  -- Набор (assetid, количество)
  comment{-- Собственность (assetid, userid владельца, количество)}
  -- Таблица предложения (assetid на assetid, userid продавца)
  -- Таблица спроса (assetid на assetid, userid покупателя)
- Операция App
  -- Journal
  --- Создать пользователя userid с уставным капиталом от 10usd и выше
  --- Удалить пользователя
  --- Зарегистрировать assetid с ликвидностью от 50usd
  --- Удалить assetid (весь asset должен принадлежать пользователю)
  --- Внести usd для userid (usd (aka доллар сша) - это тип asset)
  --- Вывести usd для userid
  --- Купить asset
  --- Продать asset
  -- Trace
  --- Соединить с биржей
  --- Получить данные с биржи
  --- Локальная проверка корректности (упреждение ошибок в ответе)
  --- Отправить запрос в биржу
  --- Получить ответ от биржи
  -- Error
  --- нет asset
  --- системная ошибка
- Операция System
  -- Trace
  --- Отправить запрос
  --- Получить ответ
  -- Error
  --- нет сети
  --- отказано в доступе

## Быстрый старт

### Базовые требования

- [Rust Toolchain](https://rust-lang.org/tools/install/) (Rust и Cargo, версия
  1.90+)
- Система контроля версий [Git](https://clck.ru/V9Qx2).

### Сборка

```shell
git clone git@github.com:Shindler7/r6sprint.git
cd r6sprint/
cargo build --release
```

### Запуск

#### Демонстрационная

```shell
target/release/cli
```

Чтобы посмотреть пример парсинга файла (например `example.log`),
его нужно поместить в текущую директорию, и передать название в качестве
аргумента:

```shell
target/release/cli example.log
```
