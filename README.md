# Парсер логов

Задача: рефакторинг предоставленного кода проекта, обеспечение работы
функциональных возможностей.

## Функционал

- парсинг log-файлов, с фильтрацией по `request_id`, ошибкам, изменению счёта
- обработка различных типов логов: `trace`, `error`, `journal`
  (человекочитаемая сводка)

## Быстрый старт

### Базовые требования

- [Rust Toolchain](https://rust-lang.org/tools/install/) (Rust и Cargo, версия
  1.90+)
- Система контроля версий [Git](https://clck.ru/V9Qx2).

### Сборка

```shell
git clone git@github.com:Shindler7/r6sprint.git
cd r6sprint/
cargo build --release
```

### Запуск

#### Демонстрационная

```shell
target/release/cli
```

Чтобы посмотреть пример парсинга файла (например `example.log`),
его нужно поместить в текущую директорию, и передать название в качестве
аргумента:

```shell
target/release/cli example.log
```

## Рефакторинг

- создана новая архитектура проекта: один неструктурированный файл `parse.rs`
  преобразован в модуль parser, и структурирован по отдельным файлам
- улучшен cli-парсинг аргументов командной строки
- убраны избыточные `RefCell` и `Rc` при работе с парсерами
- трейт парсинга обновлён, вместо `String` используется `&str`, что позволило
  убрать по коду избыточные `clone()`
- оптимизирован интерфейс api: однообразные методы заменены единым дженериком,
  убран избыточный синглтон `LOG_LINE_PARSER`
- для отдельных enum добавлен `Box<T>`, что позволило уменьшить объём данных
  на стеке
- добавлен тип ошибки `ParsersError` (под развитие), введено использование
  на последней миле `Anyhow`
- локальные улучшения кода, устранены все замечания Clippy, а также удалены
  отдельные нереализованные и (или) неиспользуемые участки кода
